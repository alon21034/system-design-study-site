Overview

What this plan is for

This plan is designed for interview candidates who need practical foundations, not a deep distributed-systems course. The goal is to build a clear answer structure, recognize coding patterns, and explain engineering trade-offs.

System design goal

Within 30–45 minutes, clarify requirements, draw a high-level design, define APIs and data models, and explain 2–3 meaningful trade-offs.

Coding goal

Practice 3–5 easy-to-medium problems per day, grouped by pattern, and review correctness, complexity, style, edge cases, and communication.

AI usage rule

Use AI to review, challenge, and ask questions. Do not use AI to immediately generate final answers before you have tried the problem yourself.

System Design

Four-week system design plan

Read selected parts of System Design Primer, then practice explaining architecture choices and trade-offs.

Week 1

Core vocabulary

Scalability, latency, throughput, availability, consistency, CAP, DNS, CDN, load balancers, and reverse proxies.

Thinking questions

  • When should a small service stay monolithic instead of being split into services?
  • Should you add a cache, a read replica, or a load balancer first? What signal justifies each option?
  • Where would CDN caching help, and where could it create freshness problems?
Week 2

Data layer, cache, and async processing

RDBMS, replication, federation, sharding, SQL vs NoSQL, cache update patterns, queues, workers, retries, and idempotency.

Thinking questions

  • What makes a good shard key?
  • When is SQL safer than NoSQL despite scaling concerns?
  • Should image processing happen synchronously or through a queue?
Week 3

Interview problem practice

Practice URL shortener, news feed, web crawler, LRU cache, and basic object-oriented design questions.

Thinking questions

  • For a URL shortener, should short codes be random or generated from an incremental ID?
  • For a feed system, when would you choose fanout-on-write, fanout-on-read, or a hybrid design?
  • For a crawler, how do you balance freshness, coverage, and politeness?
Week 4

Mock interviews and reinforcement

Practice full 45-minute mocks, back-of-the-envelope estimation, real-world architecture reading, communication, security, and AI security basics.

Thinking questions

  • Which three decisions in your final design matter the most?
  • What would you remove if cost became the top constraint?
  • Which decision would deserve an ADR?
Projects

Weekly mini projects

Week 1

Mini URL Shortener

Build APIs for creating and resolving short URLs. Include schema, collision handling, tests, and a short README.

Week 2

Image Upload + Async Worker

Model upload metadata, queue background thumbnail generation, track status, and handle retries/idempotency.

Week 3

Mini Feed / Timeline

Implement follow, post, and timeline APIs. Compare fanout-on-read and fanout-on-write.

Week 4

Final Capstone

Choose one project and polish it into a portfolio artifact with README, API spec, architecture diagram, tests, trade-offs, and AI review notes.

LeetCode

One-month coding interview plan

Practice 3–5 easy-to-medium problems per day. The focus is pattern recognition, correctness, edge cases, complexity, and interview communication.

Days 1–7

Arrays, Hash Maps, Two Pointers, Sliding Window, Stack

Practice frequency maps, sets, prefix sums, two pointers, sliding windows, and stack patterns.

  • Two Sum, Contains Duplicate, Valid Anagram, Group Anagrams
  • Product of Array Except Self, 3Sum, Container With Most Water
  • Longest Substring Without Repeating Characters, Min Stack, Daily Temperatures
Days 8–14

Binary Search, Linked List, Intervals, Matrix

Practice search invariants, dummy nodes, fast/slow pointers, sorted interval merging, and matrix traversal.

  • Search in Rotated Sorted Array, Koko Eating Bananas
  • Reverse Linked List, Remove Nth Node From End of List, Reorder List
  • Merge Intervals, Insert Interval, Spiral Matrix, Rotate Image
Days 15–21

Trees, Heap, Backtracking, Trie

Practice recursion, tree traversal, BST invariants, priority queues, decision trees, and prefix lookups.

  • Maximum Depth, Diameter, Level Order, Validate BST
  • Kth Largest Element, K Closest Points
  • Subsets, Combination Sum, Permutations, Implement Trie
Days 22–30

Graphs, Greedy, Dynamic Programming, Bit Manipulation, Mixed Mocks

Practice BFS/DFS, topological sort, greedy reasoning, state transitions, and timed mixed interviews.

  • Number of Islands, Clone Graph, Course Schedule
  • Maximum Subarray, Jump Game, Coin Change, House Robber
  • Longest Common Subsequence, Counting Bits, mixed mock sets

Documentation

Common engineering documents

Good engineers document product intent, design decisions, technical proposals, API contracts, and operational knowledge.

PRD

Defines the problem, users, goals, non-goals, requirements, and success metrics.

ADR

Records an architecture decision, alternatives considered, and consequences.

Design Doc

Explains the technical proposal, APIs, data model, trade-offs, rollout, risks, and open questions.

RFC

Used to gather comments on cross-team or long-term technical proposals.

API / Technical Spec

Defines endpoints, request/response schemas, auth, errors, rate limits, and examples.

Runbook / README

Explains how to run, deploy, monitor, debug, roll back, and troubleshoot a service.

Further Study

Topics not covered deeply in four weeks

These topics are important but intentionally left for later study so the first month stays focused.

Search systems

Inverted indexes, ranking, query parsing, reindexing, and autocomplete.

Observability

Metrics, logs, traces, SLOs, alerting, dashboards, and incident response.

Consensus and leader election

Raft, Paxos, quorum reads/writes, and distributed coordination.

Microservices platform topics

API gateways, service discovery, service mesh, circuit breakers, and service boundaries.

SQL tuning and index internals

Query planners, B-trees, covering indexes, normalization, denormalization, and EXPLAIN.

Deployment and migration

Canary rollout, blue-green deployment, feature flags, schema migration, and rollback plans.
