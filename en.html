Overview

Goals and principles

This plan is not about reading every line of System Design Primer. It focuses on the most interview-relevant concepts, answer structure, trade-off judgment, coding patterns, and documentation habits.

System design outcome

Within 30–45 minutes, clarify requirements, define APIs, sketch a high-level architecture, explain core components, and discuss trade-offs.

Coding outcome

Practice 3–5 easy-to-medium problems per day by pattern, then review correctness, complexity, style, best practices, and communication.

AI usage rule

Use AI to review, challenge, and ask questions. Do not ask AI for the final answer before you have tried the problem yourself.

AI / Vibe Coding

Additional skills in the AI era

AI can help you prototype faster, but it does not replace fundamentals. The highest-value engineering skill is still judgment: clarifying requirements, choosing trade-offs, verifying AI output, and documenting decisions.

Prompt decomposition

Break “build a service” into requirements, API, data model, testing, deployment, and review steps.

Context management

Prepare README, API specs, schemas, test results, and assumptions so AI can reason with the right context.

Verification loop

Manually read generated code, run tests, check security, validate edge cases, and review maintainability.

Trade-off judgment

When AI suggests multiple designs, evaluate cost, risk, timeline, reliability, and operational complexity.

Decision documentation

Write down why you chose option A over option B using ADRs, design docs, or RFCs.
