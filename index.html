Overview

學習目標與原則

這份計劃不要求把 System Design Primer 全部讀完，而是把面試最常用的基礎元件、回答流程、trade-off 判斷、coding patterns 與文件化能力練到能說出口。

一個月後的目標

能在 30–45 分鐘內拆解一題簡單 system design 題目，並每天完成 3–5 題 easy–medium coding practice。

每日節奏

System Design 讀 repo / 延伸閱讀；LeetCode 以 pattern 為單位練題；完成後用 AI 做 code review。

AI 使用方式

AI 用來 review 正確性、style、best practice、edge cases 與口說表達，不直接代寫最佳解。

AI / Vibe Coding

AI 時代的補充能力

AI / vibe coding 是加速學習與原型開發的工具，不是取代 system design 與 coding 基礎。工程師最大的價值，會越來越集中在需求釐清、判斷 trade-off、驗證 AI 產物與把決策寫清楚。

Prompt decomposition

把「做一個服務」拆成需求、API、資料模型、測試、部署步驟。

Context management

整理 README、API spec、schema、測試結果，讓 AI 能理解上下文。

Verification loop

AI 產生後要人工讀 code、跑測試、檢查安全與可維護性。

Trade-off judgment

對 AI 給出的多個方案，判斷成本、風險、時程、可靠性與可維護性。

Code review

讓 AI 檢查 correctness、coding style、best practice、edge cases 與複雜度。

30-Day Plan

四週讀書計劃

點擊週次可以展開/收合。每天都有 repo 閱讀、英文延伸閱讀、AI 補充、思考題與練習產出。

Thinking Questions

架構思考題

每天讀完 repo 後，回答該日卡片裡的思考題。重點不是背答案，而是把 Context、Options、Decision、Consequences 說清楚。

Documentation

常見設計文件與寫作方式

PRD、ADR、Design Doc、RFC 等文件目的不同。好的工程師要能根據決策階段選擇合適文件，讓團隊對齊「要做什麼、為什麼做、怎麼做、為什麼這樣做」。
