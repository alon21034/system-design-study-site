Overview

學習目標與原則

這份計劃不要求把 System Design Primer 全部讀完，而是把面試最常用的基礎元件與回答流程練到能說出口。

一個月後的目標

能在 30–45 分鐘內拆解一題簡單 system design 題目，畫出高階架構，說出基本 trade-off。

每日節奏

讀 repo 30–40 分鐘、延伸閱讀 15–20 分鐘、寫筆記或畫圖 20 分鐘、小練習 20–40 分鐘。

AI 使用方式

可以用 AI 加速原型、測試與 review，但要保留 prompt log、人工驗證與設計取捨。

AI / Vibe Coding

AI 時代的補充能力

AI / vibe coding 是加速學習與原型開發的工具，不是取代 system design 基礎。

Prompt decomposition

把「做一個服務」拆成需求、API、資料模型、測試、部署步驟。

Context management

整理 README、API spec、schema、測試結果，讓 AI 能理解上下文。

Verification loop

AI 產生後要人工讀 code、跑測試、檢查安全與可維護性。

Code review with AI

請 AI 找 race condition、missing index、cache invalidation、retry/idempotency。

AI security basics

認識 prompt injection、資料外洩、secret handling、權限邊界與不可信輸出。
